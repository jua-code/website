1Caracteristicas .NET

Es el encargado de proveer lo que se llama código administrado, es un entorno que provee servicios automáticos al código que se ejecuta. El Código administrado cuya ejecución está administrada mediante un tiempo de ejecución por lo cual el tiempo de ejecución en cuestión se denomina (Common Language Runtime) o CLR, independientemente de la implementación (Mono, .NET Framework o .NET Core). CLR se encarga de tomar el código administrado, compilarlo en código máquina y, después, ejecutarlo.

Permite cargar en memoria las clases.

Transforma código intermedio de alto nivel independiente del hardware que lo ejecuta a código de máquina propio del dispositivo que lo ejecuta.

Coordina toda la operación de los distintos subsistemas del Common Language Runtime.

Elimina automáticamente de memoria objetos no utilizados.

2.NET Framework.

2.1Características y sus ventajas en la plataforma

•Programación Orientada a Objetos:

La plataforma fue construida aplicando el paradigma de Programación Orientada a Objetos (POO). El núcleo de lenguajes como C# están basados en los principios OO.

•Fácil desarrollo basado en componentes:

En la plataforma es más fácil implementar componentes o bibliotecas de componentes que comparten funcionalidades. Una de esas se denomina ensamblado (assembly).

•Manejo de excepciones:

El tratamiento de errores en la plataforma se proporciona a través de un mecanismo conocido como el Error Handling. Esto elimina las comprobaciones manuales que se hacían en la programación de Win32 y basada en COM con los llamados HRESULT.

•Interoperabilidad con código existente:

La plataforma tiene muy buen soporte para reutilizar componentes de software existentes que fueron escritos por tecnologías antiguas como los componentes COM.

•Soporte para comunicación con Servicios Web:

La plataforma, ha tenido soporte para servicios web como una forma de realizar llamadas a procedimientos remotos basadas en SOAP y XML con la cual puede establecer una Arquitectura Orientada a Servicios (SOA).

•Modelo de seguridad mejorada:

El motor en tiempo de ejecución de la plataforma tiene un mecanismo de seguridad integrada llamado CAS. Este modelo de seguridad es de a los mecanismos de seguridad proporcionados por el sistema operativo, como ACL y tokens de seguridad de Windows.

•Soporte para Biblioteca de Clases Base:

La plataforma viene con un conjunto de bibliotecas de clases que proveen bloques básicos para construir aplicaciones, todas se proporcionan de manera consistente y están diseñadas por el lenguaje POO.

•Soporte para múltiples lenguajes:

La plataforma como herencia entre lenguajes (Cross-Language Interoperability) que junto con un sistema de tipos unificado (Common Type System), hace que la integración entre el código escrito en diferentes lenguajes sea total.

•Simplifica el despliegue de las aplicaciones:

En contraste con las aplicaciones basadas en componentes COM, en ello no va ser necesario. Con un “Xcopy Deployment” elimina por completo el problema de DLL HELL, gracias a las versiones de un ensamblado pueden coexistir en la misma máquina.

•Implementación de varios tipos de aplicaciones:

La Biblioteca BCL da con facilidad el poder implementar cualquier tipo de aplicación. Ya sean basadas en escritorio, aplicaciones Web, servicios Web, aplicaciones para dispositivos inteligentes, aplicaciones móviles para Windows Phone, etc.

•Infraestructura de tiempo de ejecución unificada:

En el núcleo de .NET Framework se asienta en CLR, que proporciona la infraestructura de tiempo de ejecución unificada en la forma de un lenguaje intermedio que comparten todos los lenguajes que se ejecutan en la plataforma.
